EXEMPLOS CURL - COPIE E COLE NO TERMINAL
================================================================================

1. INSCREVER NA NEWSLETTER
──────────────────────────
curl -X POST http://localhost:3001/api/newsletter/subscribe \
  -H "Content-Type: application/json" \
  -d '{"email":"usuario@exemplo.com"}'

2. CONTAR INSCRITOS
───────────────────
curl http://localhost:3001/api/newsletter/count

3. DESINSCREVER
───────────────
curl -X POST http://localhost:3001/api/newsletter/unsubscribe \
  -H "Content-Type: application/json" \
  -d '{"token":"abc123def456..."}'

4. ENVIAR MENSAGEM DE CONTATO
─────────────────────────────
curl -X POST http://localhost:3001/api/contact \
  -H "Content-Type: application/json" \
  -d '{
    "name": "João Silva",
    "email": "joao@exemplo.com.br",
    "phone": "(11) 98765-4321",
    "message": "Gostaria de saber mais sobre vossas atividades de educação em saúde. Esta mensagem possui mais de 10 caracteres."
  }'

5. LISTAR MENSAGENS DE CONTATO (ADMIN)
──────────────────────────────────────
curl "http://localhost:3001/api/contact/messages?page=1&limit=20" \
  -H "Authorization: Bearer seu_jwt_token_aqui"

6. ATUALIZAR STATUS DE MENSAGEM (ADMIN)
────────────────────────────────────────
curl -X PATCH http://localhost:3001/api/contact/42 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer seu_jwt_token_aqui" \
  -d '{"status":"replied"}'

7. INSCREVER EM EVENTO
──────────────────────
curl -X POST http://localhost:3001/api/events/42/subscribe \
  -H "Authorization: Bearer seu_jwt_token_aqui"

8. CANCELAR INSCRIÇÃO EM EVENTO
───────────────────────────────
curl -X DELETE http://localhost:3001/api/events/42/subscribe \
  -H "Authorization: Bearer seu_jwt_token_aqui"

9. LISTAR INSCRITOS EM EVENTO (ADMIN)
──────────────────────────────────────
curl "http://localhost:3001/api/events/42/inscriptions?page=1&limit=50" \
  -H "Authorization: Bearer seu_jwt_token_aqui"

10. OBTER ESTATÍSTICAS DO SITE
───────────────────────────────
curl http://localhost:3001/api/website/stats

TABELAS CRIADAS NO MYSQL (4/4)
================================================================================

1. newsletter_subscribers
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - email (VARCHAR 255, UNIQUE)
   - dateSubscribed (DATETIME)
   - isActive (BOOLEAN, DEFAULT true)
   - unsubscribeToken (VARCHAR 64, UNIQUE)
   - createdAt (DATETIME)
   - updatedAt (DATETIME)

   Índices: email, isActive, unsubscribeToken

2. contact_messages
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - name (VARCHAR 255)
   - email (VARCHAR 255)
   - phone (VARCHAR 20)
   - message (TEXT)
   - status (ENUM: new, read, replied, archived, DEFAULT 'new')
   - createdAt (DATETIME)
   - updatedAt (DATETIME)

   Índices: email, status, createdAt

3. event_inscriptions
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - userId (INT, FOREIGN KEY -> app_users.id)
   - eventId (INT, FOREIGN KEY -> health_events.id)
   - status (ENUM: registered, confirmed, cancelled, DEFAULT 'registered')
   - checkedInAt (DATETIME)
   - createdAt (DATETIME)
   - updatedAt (DATETIME)

   Constraint: UNIQUE(userId, eventId)
   Foreign Keys: ON DELETE CASCADE
   Índices: status, userId, eventId, createdAt

4. blog_categories
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - name (VARCHAR 100, UNIQUE)
   - slug (VARCHAR 100, UNIQUE)
   - color (VARCHAR 7, DEFAULT '#6366f1')
   - description (TEXT)
   - createdAt (DATETIME)
   - updatedAt (DATETIME)

   Índices: slug
